# Ad copy library — Responsive Search Ads

One RSA per ad group. Each RSA: 12-15 headlines, 4 descriptions, all four account-level extension types (sitelinks, callouts, structured snippets, call/location/lead-form).

Files:
- [`rsas.md`](rsas.md) — every RSA, ready to copy/paste into Google Ads Editor or the UI.
- [`assets.md`](assets.md) — account-level assets (sitelinks, callouts, structured snippets) shared across all campaigns.

## Headline character limits

- Headlines: up to **30 characters** each. Every headline in [`rsas.md`](rsas.md) has been kept ≤ 30. If you change copy, [count characters](https://wordcounter.net/character-count) before saving.
- Descriptions: up to **90 characters** each. Same.
- Final URL: always the matched landing page (see Section 10.8 in `../PHASE-10-GOOGLE-ADS.md`).
- Display path: use the per-LP slug, e.g. `cdhcranerentals.com/Lafayette` or `/Operated` or `/Emergency`.

## Pin / unpin

- **Pin position 1** on **at least one** headline that contains the city or service for high-message-match queries. Example: `Crane Rental in Lafayette, LA` pinned to position 1 for the Lafayette · Generic ad group.
- Do not over-pin; Google's algorithm tests combinations.

## Quality Score expectations

With message match this tight (keyword theme ↔ ad headline ↔ LP H1), expect QS 8–10 across the board within 30 days, assuming LP CWV stays "Good".
